## 已修复的 Bug

### 1. SPECIAL_LINK_PATTERNS 重复声明
**问题**: secrets.js 和 main.js 都用 `const` 声明了同一个变量
**修复**: secrets.js 改为 `window.SPECIAL_LINK_PATTERNS`，main.js 从 window 获取

### 2. formatSecretWithLinks 语法错误
**问题**: 孤立的数组定义导致语法错误
```javascript
// ❌ 错误
const SPECIAL_LINK_PATTERNS = window.SPECIAL_LINK_PATTERNS || {};
{ type: 'organization', regex: ... },
];
```
**修复**: 删除孤立数组，使用 `for...of Object.entries()` 遍历对象

### 3. showLoreModal 数据结构不匹配
**问题**: lore-data.js 导出数组，但 showLoreModal 用对象方式访问
```javascript
// ❌ 错误
loreData = LORE_ORGANIZATIONS[name];
```
**修复**: 改用 `.find()` 方法
```javascript
// ✅ 正确
loreData = LORE_ORGANIZATIONS.find(org => org.name === name);
```

### 4. 特殊链接 CSS 样式缺失
**问题**: `.secret-link` 和 `.lore-link` 没有定义样式，链接看起来像普通文本
**修复**: 添加蓝色虚线下划线 + hover 效果

## ⚠️ 未解决的关键问题

### 正则表达式无法匹配中文
**现象**: SPECIAL_LINK_PATTERNS 中的正则无法匹配 `[非常电影院]` 等中文标记

**测试代码**:
```javascript
const regex = /\[([^\]]*?(?:黎守 | 对伪课 | 非常电影院))\]/g;
const text = '[非常电影院] 深夜放映时';
console.log([...text.matchAll(regex)]); // 输出：[]
```

**可能原因**:
1. Node.js 正则对中文或运算支持问题
2. 正则中的空格导致匹配失败
3. 需要 Unicode 标志 `u`

**待验证**: 在浏览器环境中测试（可能浏览器 JS 引擎与 Node.js 行为不同）

**临时解决方案**: 简化正则，去掉 `[^\]]*?` 前缀，直接匹配关键词
```javascript
organization: /\[(黎守 | 对伪课 | 非常电影院)\]/g
```

## 图片加载优化（已完成）

### 策略演进
1. **初始**: IntersectionObserver 懒加载（滚动到视口才加载）
2. **问题**: 用户反馈"点到别的页面图片也没加载出来"
3. **最终方案**: 预加载所有图片，但分优先级

### 最终实现
```javascript
class ImagePreloader {
    // 高优先级：视口内 + 前 3 屏图片 → 直接加载
    // 低优先级：更远的图片 → 后台加载
    // 最大并发：6 张图片
}
```

### 优化效果
- 首屏图片快速加载
- 远处图片后台加载，不阻塞首屏
- 滚动时图片已经加载完成，无等待感
