## 🚨 血泪教训：绝不覆盖用户微调过的文件！（2026-04-03）

### 事故经过
用户花了 **40+ 轮对话**微调导航页面（梦核风格：深青绿、CRT扫描线、噪点、毛玻璃、状态栏、水印暗纹、访客计数器、随机文字碎片...），我运行 `generate-nav.sh` 添加新子目录时，脚本用旧版简单模板**覆盖整个 index.html**，所有微调全部丢失。

### 根本原因
`generate-nav.sh` 内嵌完整 HTML 模板，运行时会**重写整个文件**，而不是只增删卡片。

### 铁律（必须遵守！）

1. **🔴 绝不运行会覆盖整个文件的脚本**
   - 如果脚本会重写 index.html，**绝对不要运行**
   - 用户微调过的文件 = 资产，必须保护

2. **✅ 添加新子目录的正确方式**
   - 直接在 index.html 的卡片区域手动插入新卡片
   - 用 `edit` 工具精确定位，只改那一块
   - 运行前**先备份**：`cp index.html index.html.bak`

3. **✅ generate-nav.sh 的正确角色**
   - 脚本应该只备份 + 扫描目录 + 提示用户
   - 不应该生成或覆盖 index.html
   - 已经修复为安全版本

4. **✅ 通用原则**
   - 用户花的每一轮微调都是时间成本
   - "动态更新"≠"覆盖重建"
   - 宁可手动改，不要冒险覆盖
   - **改之前先备份，改之后确认没丢东西**

- 每次修改前自动创建 `.pre-nav-update` 备份
